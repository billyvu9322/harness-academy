# Harness Documentation Rules

- Keep test plans reviewable by testers, not only by developers.
- Every scenario needs preconditions, steps, expected results, locator strategy, and meaningful assertions.
- Evidence reports must include command, result, artifacts, failure attribution, and human review decision.
- If behavior changes, update the approved test plan before changing test expectations.
